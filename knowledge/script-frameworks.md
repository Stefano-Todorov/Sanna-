# Script Frameworks — Structures That Keep Viewers Watching

Every high-performing short-form video follows a structure. These frameworks ensure the viewer stays through the entire video (watch time is king).

## Primary Frameworks

### AIDA — Attention → Interest → Desire → Action
The classic marketing framework adapted for short-form.
- **Attention (0-3s):** Hook that stops the scroll
- **Interest (3-15s):** Why this matters to THEM specifically
- **Desire (15-45s):** Show the outcome, proof, or transformation they want
- **Action (last 3-5s):** Clear CTA — what to do next
**Best for:** Product content, tutorials, value-driven posts
**Typical length:** 30-60 seconds

### PAS — Problem → Agitate → Solution
Identify a pain point, make it feel urgent, then deliver the fix.
- **Problem (0-5s):** Name the specific pain point (hook)
- **Agitate (5-20s):** Twist the knife — show WHY it's worse than they think, what it's costing them
- **Solution (20-45s):** Deliver your answer with specifics
**Best for:** Pain-point content, advice, "stop doing X" videos
**Typical length:** 30-60 seconds

### Storytelling Arc
Mini three-act structure for narrative content.
- **Setup (0-5s):** Drop into the action — start at the most dramatic moment
- **Rising tension (5-30s):** Escalate — each sentence should raise stakes
- **Resolution (30-45s):** Payoff — the lesson, reveal, or twist
- **Tag (last 3s):** Quick CTA or callback to the hook
**Best for:** Storytime, case studies, personal experiences
**Typical length:** 45-90 seconds

### List Framework
Structured, scannable content with clear progression.
- **Promise (0-3s):** "X things that [benefit]" — set the number
- **Items (3-40s):** Each item gets 5-10 seconds. Escalate in value — save the best for last
- **Closer (last 5s):** Reinforce the best point + CTA
**Best for:** Tips, mistakes, rules, rankings
**Typical length:** 30-60 seconds
**Key rule:** Number in the hook MUST match delivered items. Never promise 5 and give 4.

### Tutorial / How-To
Step-by-step instruction with visual demonstration.
- **Result first (0-3s):** Show the end result immediately (before/after)
- **Steps (3-45s):** Walk through each step clearly. Use text overlays for each step number
- **Recap (last 5s):** Quick summary + CTA to save
**Best for:** Educational content, DIY, recipes, processes
**Typical length:** 30-90 seconds

## Pacing Rules for Short-Form

1. **New visual or info every 3-5 seconds** — if nothing changes for 5+ seconds, viewers drop off
2. **Front-load the value** — don't "save the best for last" unless the hook promises it
3. **Escalate, don't plateau** — each section should be more interesting than the last
4. **The body is NOT filler** — every sentence should either teach, entertain, or advance the story
5. **Transitions matter** — "but here's the thing", "and it gets worse", "now watch this" keep momentum
6. **End strong, not abruptly** — the CTA should feel like a natural conclusion, not a cutoff

## Script Length Guidelines

| Difficulty | Duration | Word Count | Sections |
|---|---|---|---|
| Easy (talking head) | 30-60s | 75-150 words | Hook + 1-2 points + CTA |
| Medium (some edits) | 45-90s | 110-225 words | Hook + 2-3 points + CTA |
| Hard (full production) | 60-180s | 150-450 words | Hook + 3-5 points + transitions + CTA |

## Body Writing Rules

1. **Write for speaking, not reading** — use contractions, fragments, casual language
2. **One idea per sentence** — complex sentences lose viewers
3. **Use section markers** — [INTRO], [MAIN POINT], [B-ROLL], [TEXT OVERLAY] help creators film
4. **Include visual cues** — describe what should be on screen, not just what to say
5. **Vary sentence length** — rhythm matters. Short punch. Then a longer explanation that builds on it.
